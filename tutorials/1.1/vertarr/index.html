Vertex Arrays & Custom rendering

What is a vertex array?

A vertex array is, simply put, an array of vertices. You can bind your own functions to the shape module and the WindowRenderer object to create your own shapes and objects, for example, a video player.

Vertex Arrays

Vertex arrays don't have their own class as technically they're just a shape; which is why you use the shape class to make it - You can use the vertexArray function to create a vertex array.
local clearscreen = shape.vertexArray("custom_clearscreen", function(Color): { Command: string, Data: { [any?]: any? } } return { Command = "custom_clearscreen", Data = { Color = Color } } end, function(Window, self) local arguments = self.Args.Object.Data local RGB, Alpha = color.toRGB(arguments.Color) print("The screen was cleared!") Window.Renderer:DrawRectangle(Vector2.zero, Window.Renderer.Size, RGB, Alpha) end) myWindow:draw(shape.custom_clearscreen(color.Blue)) myWindow:render()


I know the above code does look scary, but don't worry. I'll explain it. Before I get into it, since these functions are registered at the core of OSGL (WindowRenderer), it's best to have some knowledge of how EditableImages work since calling Render in this case won't work - You'll have to use the functions the EditableImage API provides you with.

The code creates a version of clear that prints "The screen was cleared!" every time the screen is cleared. The first argument is the name of our new vertex array. In this case, it's "custom_clearscreen". We then pass 2 functions.

The first function is the one that will be appended to the shape class - this means you can override the existing inbuilt functions. The function must return a "RenderCommand" - This is what the argument buffer gives to the WindowRenderer. It must have the command you want the argument buffer to call (In this example, "custom_clearscreen"), and the arguments that you want to pass in a "Data" array.

The second function is the one that is attatched to the WindowRenderer. It is given the Window, and "self". "self" is the value that was returned in the first function, but there are a few tables appended around it. self.Name is the name of the command being run, and self.Args.Object.Data is the array of arguments that we passed.

In the example we gave, we convert the RGBA Color to a Color3 and Alpha value, and draw a rectangle onto the EditableImage. Finally, the function is called from the shape class with a blue color and the screen is rendered once - making the window appear blue.
