Sprites & Textures

What are they?

Most of you reading this are probably familiar with these 2 objects that are very common in game development - but lets define them very briefly.

A texture is essentially an OSGL Image. We call it a "texture" because it has one specific role: being mapped to a 2d entity on the screen.

A sprite is nothing more than a textured rectangle. It is simply a rectangle with a texture overlapped on top of it.

Loading textures

Before creating any sprites, we need textures to map to them. The class that allows the creation of textures in OSGL, is surprisingly, believe it or not, texture! Since the only role of a texture is to be mapped to a sprite, all this is, is, a simple constructor for us.

The most common way to load a texture is from a module - this can be done with the loadFromModule function:
local OSGL = require(script.OSGL) local window = OSGL.window local enum = OSGL.enum local color = OSGL.color local texture = OSGL.texture local myWindow = window.new({ Size = enum.WindowSize.Maximum, Instance = script.Parent })() local myTexture = texture.loadFromModule(script.PlayerTexture) while myWindow:isOpen() do myWindow:clear(color.Black) --// ... myWindow:render() end


You can also load a texture directly from a string with (you'll never guess this): loadFromString! Both functions come with an extra argument which can be used if you want to crop a part of the texture:
local OSGL = require(script.OSGL) local window = OSGL.window local enum = OSGL.enum local color = OSGL.color local texture = OSGL.texture local myWindow = window.new({ Size = enum.WindowSize.Maximum, Instance = script.Parent })() -- Crop a 32x32 rectangle from 10, 10. local myTexture = texture.loadFromModule(script.PlayerTexture, texture.rectCrop(Vector2.new(10, 10), Vector2.new(32, 32))) while myWindow:isOpen() do myWindow:clear(color.Black) --// ... myWindow:render() end


The rectCrop function is a simple utility function which represents a rectangle. It takes the coordinates of the top-left corner, and the size of the rectangle.

Currently, textures cannot be edited at runtime, but they can be created. OSGL provides a multitude of functions for creating a texture during runtime. Textures are made with the texture.new function. An "EditableTexture" has the ability for its textures to be modified in memory - it cannot be displayed until the finish method is called, in which it will return the final copy of the image in OSGL format (as a string).

Okay.. but like.. can I have my sprite now?

Yes; you can have your sprite now.

local OSGL = require(script.OSGL) local window = OSGL.window local enum = OSGL.enum local color = OSGL.color local texture = OSGL.texture local sprite = OSGL.sprite local myWindow = window.new({ Size = enum.WindowSize.Maximum, Instance = script.Parent })() local myTexture = texture.loadFromModule(script.PlayerTexture) local mySprite = sprite.new(myTexture) while myWindow:isOpen() do myWindow:clear(color.Black) myWindow:draw(mySprite) myWindow:render() end


Sprites have numerous properties that allow us to change how it's rendered. We can change its:

  • Texture (Texture)
  • Color (number)
  • Position (Vector2)
  • Scale (Vector2)


Sprites are, of course, drawn with windows' draw method. Just pass the sprite through the function to add it to the argument buffer.

The importance of using as little textures as possible

Using as few textures as possible is a good strategy, as changing textures is an expensive operation for Roblox, and the graphics card. Drawing many sprites with the same texture will give the best performance - You can have multiple textures using sprite-maps. Keep this in mind while creating textures - remember to create tilesets and animation sheets to use as little textures as possible.
